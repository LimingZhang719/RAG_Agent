# P1 基础后端与数据模型 - 研发计划

> 目标：在 P0 基础可运行的前提下，完成核心数据模型、数据库迁移、基础鉴权与种子数据，为 P2-P6 打好后端基础。

## 1. 范围与交付物

### 1.1 本阶段范围

- 数据库：PostgreSQL + SQLAlchemy ORM + Alembic 迁移。
- 核心数据模型：用户、组织、角色、知识库、文档、聊天、检索日志、Agent 运行、报销相关。
- 后端分层：api router -> service -> repository/model -> external adapter。
- 基础鉴权与 RBAC：本地用户、角色、权限校验。
- 种子数据：管理员、普通用户、财务用户、示例组织。

### 1.2 交付物清单

- 完整 SQLAlchemy 模型（含关系、索引、枚举）。
- 初始 Alembic migration。
- DB session 管理与基础 repository。
- 基础 RBAC 与用户登录最小接口（本地账号）。
- Seed 脚本与示例数据。
- 基础模型单元测试。

## 2. 目标验收标准

- migration 可从空库成功创建全部表。
- seed 后可登录不同角色用户。
- 单元测试覆盖模型创建和基础查询。
- 关键表具备必要索引与唯一约束。

## 3. 依赖与前置条件

- P0 已完成：后端工程骨架、配置、日志、异常处理、Alembic 初始化。
- PostgreSQL + pgvector 已可用。
- `.env` 已包含数据库连接配置。

### 4.2 关键字段与约束

- chunks
  - 需要 pgvector 字段
  - 需要权限冗余字段（visibility_scope/org_id/owner_id）
  - 需要 `is_deterministic_rule`
- user_roles
  - user_id + role_id 唯一
- knowledge_bases
  - name + org_id + visibility_scope 唯一（视业务而定）
- documents
  - status 枚举（pending/parsing/chunking/embedding/ready/failed）
- chat_messages
  - session_id + created_at 索引

## 5. 技术方案与实现细节

### 5.1 模型层

- 使用 SQLAlchemy 2.0 风格定义模型。
- 统一 Base 与 timestamp mixin（created_at, updated_at）。
- 引入软删除字段（可选，视当前项目实践）。
- 枚举统一放置在 models 或 core/constants。

### 5.2 数据库迁移

- 初始迁移一次性创建所有核心表。
- pgvector 扩展启用：
  - migration 中 `CREATE EXTENSION IF NOT EXISTS vector`。
- 索引：
  - chunks embedding 使用 ivfflat 或 hnsw（后续 P3 决定，当前可先预留）。

### 5.3 RBAC 与用户

- users 表：支持本地账号密码（hash 存储）。
- roles 表：admin/department_admin/user/finance。
- user_roles 表：多对多。
- 组织关系：users 归属 organizations。

### 5.4 Repository/Service

- Repository 仅做 CRUD 和 query 组合。
- Service 处理业务逻辑、事务与权限校验。

## 6. 开发任务拆解

### 6.1 数据库与模型

1. 设计并实现所有模型与关系。
2. 添加必要索引与唯一约束。
3. 完成 Alembic migration。

### 6.2 RBAC 与鉴权

1. 用户模型与密码 hash。
2. 简易登录接口（token 或 session）。
3. 权限校验工具函数。

### 6.3 Seed 与测试

1. Seed 脚本生成组织与用户。
2. 基础模型单元测试。

## 7. 里程碑与时间计划

- M1：模型设计与字段冻结。
- M2：完成 ORM 模型与 migration。
- M3：完成 RBAC 与登录接口。
- M4：完成 seed 与单测。

## 8. 风险与对策

- 风险：字段设计后期频繁变更
  - 对策：先冻结核心字段，再扩展可选字段。
- 风险：权限逻辑遗漏导致越权
  - 对策：在 service 层统一校验，并增加测试。

## 9. 验收清单（提交前自检）

- 迁移可在空库执行成功。
- Seed 后至少 4 类角色可登录。
- 单测能覆盖模型创建与基本查询。
- 关键表有索引与唯一约束。
