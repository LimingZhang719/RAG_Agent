# P2 前端基础框架 研发计划与开发细节

本文档基于：
- v0开发计划（P2 前端基础框架）
- v0版本技术设计（前端工程结构与能力范围）
- 需求文档（角色权限与页面范围）

目标：在不引入额外架构风险的前提下，完成登录、布局、路由、权限菜单、API 封装，并提供可扩展的组件与状态管理骨架。

## 1. 研发范围与交付物

### 1.1 交付物清单
- React + TypeScript + Vite 前端工程骨架（已存在，补齐缺失部分）
- 路由与页面结构：登录页、首页、知识库管理、问答、个人空间、智能体中心、报销、财务审批、管理页（先空壳页面）

### 1.2 不在本次范围

- 真实业务页面交互细节（本阶段只做骨架与占位）
- RAG/Agent 真实对话逻辑

## 2. 技术选型与约束

- React + TypeScript + Vite（现有工程）
- UI：Ant Design
- 路由：React Router
- 请求管理：React Query
- 状态：Zustand
- API 层：Axios 或 fetch（若现有依赖固定则沿用）
- 鉴权：Token 存储在 localStorage（v0 先用，后续可替换）
- 代码风格：ESLint + Prettier（如已配置则沿用）

## 5. 关键设计细节

### 5.1 路由与权限
- `ProtectedRoute`：未登录跳转登录页
- `RoleGuard`：无权限时跳转 403 页面（或重定向首页）
- 菜单与路由使用统一配置表

### 5.2 用户与角色模型
- `User`：id、name、email、roles、orgs
- `Role`：admin | department_admin | user | finance
- `Permission`：通过 role 推导，前端只做 UI 过滤

### 5.3 API Client
- 请求头自动注入 token
- 统一响应处理
- 统一错误提示
- 请求超时与重试策略（可选）

### 5.4 状态管理
- `authStore`：token、isAuthenticated
- `userStore`：currentUser、roles、orgs
- 通过 React Query 缓存 `/api/me`

- Layout 与路由稳定
