# 一键启动/停止（Windows 开发）

本文档提供 Windows + PowerShell 的一键启动/停止方式，适用于本地开发。

## 启动（一次性命令）

1. 在项目根目录执行，启动依赖服务：

```powershell
docker compose -f deploy/docker-compose.dev.yml up -d
```

2. 后端安装依赖与数据库迁移
```powershell
cd backend
pip install -e .
alembic upgrade head
```

3. 启动后端API
```powershell
cd backend
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

4. 启动 Celery Worker（文档入库任务）
 ```powershell
cd backend
celery -A app.workers.celery_app.celery_app worker --loglevel=info --pool=threads --concurrency=4
```

5. 启动前端
 ```powershell
cd frontend
npm install
npm run dev
```


说明：
- 第一次运行会创建 `.env` 并启动 PostgreSQL/Redis/MinIO。
- 后端会自动安装依赖并执行数据库迁移。
- 前端会自动安装依赖并启动 Vite。
- 两个 `Start-Process` 会打开新终端窗口（保持可见日志）。

## 停止（一次性命令）

在项目根目录执行：

```powershell
docker compose --env-file .env -f deploy/docker-compose.dev.yml down
```

说明：
- 前后端进程请在各自终端窗口按 `Ctrl+C` 停止。
- 如需强制关闭，可在任务管理器结束 `uvicorn`/`node` 进程。

## 访问地址

- 后端健康检查：`http://127.0.0.1:8000/health`
- 后端数据库健康检查：`http://127.0.0.1:8000/health/db`
- 前端页面：`http://127.0.0.1:5173`

# 一键启动/停止（Linux 服务器，Docker Compose）

本文档提供 Linux 服务器通过 Docker Compose 的一键启动/停止方式（基础设施）。

## 启动（一次性命令）

在项目根目录执行：

```bash
cp -n .env.example .env
docker compose --env-file .env -f deploy/docker-compose.linux.yml up -d
```

说明：
- 该 Compose 目前只包含 PostgreSQL/Redis/MinIO。
- 后端与前端容器化未包含在此文件中，需按你的部署方案单独启动。

## 停止（一次性命令）

在项目根目录执行：

```bash
docker compose --env-file .env -f deploy/docker-compose.linux.yml down
```
