# 用户注册功能研发计划（v0）

> 目标：新用户进入系统先到首页，首页提供登录/注册入口；注册需选择 4 类用户角色并选择组织/部门；注册成功自动登录进入首页；登录态持久化，避免重复登录。

---

## 1. 需求依据与约束

- 统一鉴权登录与权限路由渲染流程：用户登录后基于身份与组织信息渲染可见知识库。
- 角色范围：`admin`、`department_admin`、`user`、`finance`。
- 登录后可见范围与权限限制基于角色与组织。

---

## 2. 业务流程与用户体验

### 2.1 新用户进入首页

1. 未登录用户访问系统，进入首页。
2. 首页提供登录/注册入口，支持切换。
3. 注册成功后自动登录并进入首页。
4. 登录态持久化，刷新页面后仍保持登录状态。

### 2.2 注册表单与校验

- 必填字段：`username`、`password`、`role`、`org_id`。
- 选填字段：`email`、`full_name`。
- 校验规则：
  - `username`、`email` 唯一性校验。
  - `role` 必须为 4 个角色之一。
  - `org_id` 必须为有效组织/部门。
  - 密码强度暂无额外要求。

- 组织/部门候选项（固定）：
  - 办公室
  - 组织部
  - 党建
  - 财务
  - 计划部
  - 科技部
  - 主网中心
  - 配网中心
  - 自控中心
  - 设计中心
  - 技经中心汇总
  - 数据中心
  - 管评中心
  - 评审中心
  - 监造中心标注

### 2.3 自动登录与持久化

- 注册成功后返回 `access_token` 并写入持久化存储。
- 前端刷新后自动调用 `/auth/me` 恢复用户状态。
- Token 过期或失效时自动登出并提示重新登录。

### 2.4 审核流程

- 若注册角色为 `department_admin`，需要 `admin` 审核通过后生效。
- 若注册角色为 `user`，需要对应部门的 `department_admin` 审核通过后生效。
- 需要审核时，注册接口返回“待审核”状态，不下发 token；前端提示审核中。

---

## 3. API 设计与数据契约

### 3.1 注册接口

- `POST /auth/register`
- Request Body:

```json
{
  "username": "string",
  "password": "string",
  "email": "string | null",
  "full_name": "string | null",
  "role": "admin | department_admin | user | finance",
  "org_id": "uuid"
}
```

- Response Body（无需审核时）:

```json
{
  "access_token": "string",
  "token_type": "bearer",
  "expires_at": "datetime"
}
```

- Response Body（需审核时）:

```json
{
  "status": "pending_approval"
}
```

- Error Codes:
  - `409` 用户名或邮箱已存在。
  - `400` 字段校验失败。
  - `404` 组织/部门不存在。

### 3.2 组织/部门列表接口（如需）

- `GET /organizations`
- 返回组织/部门列表用于注册选择。

---

## 4. 数据模型与权限关联

- `users`：写入 `org_id` 与基础信息。
- `roles`：角色表包含 4 类角色。
- `user_roles`：注册时写入用户角色关联。

---

## 5. 前端改造范围

- 首页：新增登录/注册入口与切换。
- 注册页面：新增组织/部门选择、角色选择组件与校验提示。
- API Client：新增 `register()` 方法。
- Store：注册成功后写入 token 与用户信息，刷新后自动恢复。

---

## 6. 后端改造范围

- `auth` 路由新增注册接口。
- `auth_service`：实现注册逻辑（校验、哈希、写库、角色绑定）。
- `schemas`：新增注册请求/响应模型。
- 权限相关：确保 `org_id` 与 `role` 绑定正确。

---

## 7. 测试与验收标准

### 7.1 功能验收

- 无需审核时注册成功后自动登录并进入首页。
- 需审核角色注册后进入“待审核”提示页或状态提示。
- 组织/部门必选，未选不可提交。
- 角色必须为 4 类之一。
- 刷新页面后仍保持登录状态。
- 退出登录后无法访问受保护页面。

### 7.2 异常与边界

- 重复用户名/邮箱提示明确。
- 组织/部门不存在时返回错误。
- Token 过期后自动登出。
