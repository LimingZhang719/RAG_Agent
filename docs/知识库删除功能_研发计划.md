# 知识库删除功能研发计划

> 目标：补齐知识库管理界面的删除能力，支持删除知识库以及知识库内单个文件，并保证数据库、向量切片、对象存储与前端状态一致。

---

## 1. 背景与现状

当前知识库页面已支持：

- 创建知识库。
- 上传文档。
- 查看文档入库状态。
- 查看文档切片。
- 修改文档切块配置并重新入库。

当前缺口：

- 前端知识库列表没有删除知识库入口。
- 前端文档列表没有删除文件入口。
- 前端 API 封装缺少 `deleteKnowledgeBase()` 与 `deleteDocument()`。
- 后端已有 `DELETE /knowledge-bases/{kb_id}` 基础接口，但删除逻辑只删除 `knowledge_bases` 记录，未显式处理文档、切片、ACL、对象存储文件等关联资源。
- 后端文档模块暂无 `DELETE /documents/{doc_id}` 接口。
- MinIO 封装暂无删除对象方法。

---

## 2. 功能目标

### 2.1 删除知识库

- 用户可在知识库列表中删除有权限管理的知识库。
- 删除前弹出二次确认，明确提示该操作会删除知识库下所有文件、切片和索引数据。
- 删除成功后：
  - 知识库列表刷新。
  - 当前选中知识库若被删除，自动切换到下一条可见知识库；若无知识库，则清空文档列表。
  - 给出成功提示。
- 删除失败时展示明确错误提示。

### 2.2 删除知识库内文件

- 用户可在文档列表中删除有权限管理的文件。
- 删除前弹出二次确认，提示会删除文件、解析块、向量切片和对象存储原文件。
- 删除成功后刷新当前知识库文档列表。
- 删除失败时展示明确错误提示。

---

## 3. 权限规则

沿用现有知识库管理权限：

- `admin` 可删除所有可管理知识库及其文件。
- 非 `admin` 用户只能删除自己创建的知识库。
- 删除文件时，先校验文件所属知识库，再复用知识库管理权限：
  - `admin` 可删除任意知识库文件。
  - 知识库 owner 可删除该知识库下文件。
  - 仅有可读权限的用户不可删除。

建议将“是否可管理知识库”的判断抽成服务层函数，避免更新、删除知识库和删除文档各自写一套权限判断。

## 4. 后端研发范围

### 4.1 API 设计

#### 删除知识库

- 复用现有接口：`DELETE /knowledge-bases/{kb_id}`
- Response:

```json
{
  "status": "ok"
}
```

#### 删除文档

- 新增接口：`DELETE /documents/{doc_id}`
- Response:

```json
{
  "status": "ok"
}
```

### 4.2 服务层调整

#### `knowledge_base_service`

- 增强 `delete_knowledge_base()`：
  - 校验知识库是否存在。
  - 校验管理权限。
  - 查询知识库下所有文档及 `file_uri`。
  - 删除关联数据：
    - `chunks`
    - `document_blocks`
    - `documents`
    - `kb_acl`
    - `knowledge_bases`
  - 删除对象存储中的原文件。
  - 提交事务。

#### `document_service`

- 新增 `delete_document()`：
  - 查询文档。
  - 校验所属知识库管理权限。
  - 删除 `chunks`。
  - 删除 `document_blocks`。
  - 删除 `documents`。
  - 删除 MinIO 原文件。
  - 提交事务。

### 4.3 对象存储调整

#### `MinioStorage`

- 新增删除方法：

```python
def remove_object(self, object_name: str) -> None:
    ...
```

- 基于 `parse_minio_uri(file_uri)` 获取对象名。
- 对象不存在时建议按幂等处理：
  - 数据库删除继续完成。
  - 记录日志或返回可追踪错误。

### 4.4 数据一致性策略

推荐采用“数据库删除为主、对象存储删除尽力而为”的策略：

- 数据库事务内删除关系数据，避免残留可检索切片。
- 对象存储删除失败时记录错误并返回失败，避免用户误以为物理文件已删除。
- 如果后续需要更高可靠性，可引入异步清理任务和待清理对象表。

### 4.5 迁移评估

当前模型外键未配置数据库级 `ON DELETE CASCADE`，短期可在服务层显式删除关联数据，不强制新增迁移。

如后续希望由数据库保证级联删除，可新增 Alembic 迁移，为以下外键补充级联策略：

- `documents.kb_id -> knowledge_bases.id`
- `document_blocks.document_id -> documents.id`
- `chunks.document_id -> documents.id`
- `chunks.kb_id -> knowledge_bases.id`
- `kb_acl.kb_id -> knowledge_bases.id`

本次建议先服务层显式删除，降低迁移风险。

## 5. 前端研发范围

### 5.1 API 封装

#### `frontend/src/api/kb.ts`

- 新增：

```ts
export async function deleteKnowledgeBase(kbId: string): Promise<void> {
  await apiClient.delete(`/knowledge-bases/${kbId}`);
}
```

#### `frontend/src/api/documents.ts`

- 新增：

```ts
export async function deleteDocument(docId: string): Promise<void> {
  await apiClient.delete(`/documents/${docId}`);
}
```

### 5.2 知识库列表交互

在知识库列表新增“操作”列：

- 删除按钮。
- 点击删除按钮时阻止触发表格行选中。
- 使用 `Modal.confirm` 或 `Popconfirm` 二次确认。
- 删除中禁用按钮或展示 loading。

删除成功后的状态处理：

- 重新加载知识库列表。
- 如果删除的是当前选中知识库：
  - 优先选中刷新后第一条知识库。
  - 若列表为空，设置 `selectedKb = null`，并清空 `documents`。

### 5.3 文档列表交互

在文档列表“切片/操作”区域增加删除按钮：

- 删除按钮与“查看”“切块配置”并列。
- 删除前二次确认。
- 删除成功后刷新当前知识库文档列表。
- 如正在查看该文档切片，删除后关闭切片 Drawer 并清空 `activeDocument`、`chunkItems`。

### 5.4 前端权限展示

第一阶段可以先展示删除按钮，由后端做最终权限校验。

优化方案：

- 根据当前用户角色与 `owner_id` 判断是否显示删除按钮。
- 对无权限用户隐藏或禁用删除按钮。
- 仍保留后端权限校验作为最终保障。

## 6. 异常与边界场景

- 删除不存在的知识库：返回 `404`。
- 删除无权限知识库：返回 `403`。
- 删除不存在的文档：返回 `404`。
- 删除无权限文档：返回 `403`。
- 删除知识库时包含入库中的文档：
  - 第一阶段允许删除，同时删除已生成的块和切片。
  - Worker 后续执行时若找不到文档，应安全失败或跳过。
- 删除对象存储文件失败：
  - 返回明确错误提示。
  - 日志记录 `file_uri` 与异常信息。
- 连续点击删除：
  - 前端按钮 loading 防重入。
  - 后端删除接口保持幂等倾向，已删除资源返回 `404` 可接受。

## 7. 测试计划

### 7.1 后端测试

- `admin` 可删除任意知识库。
- 普通用户可删除自己创建的个人知识库。
- 普通用户不可删除他人知识库。
- 删除知识库后：
  - `knowledge_bases` 记录不存在。
  - 关联 `documents` 不存在。
  - 关联 `document_blocks` 不存在。
  - 关联 `chunks` 不存在。
  - 关联 `kb_acl` 不存在。
- 删除文档后：
  - `documents` 记录不存在。
  - 关联 `document_blocks` 不存在。
  - 关联 `chunks` 不存在。
  - 同知识库下其他文档不受影响。
- MinIO 删除方法被调用，并能处理对象不存在场景。

### 7.2 前端测试

- 知识库列表展示删除入口。
- 删除知识库前有确认弹窗。
- 删除当前选中知识库后，文档列表状态正确刷新。
- 删除最后一个知识库后，文档列表清空。
- 文档列表展示删除入口。
- 删除文档后当前列表刷新。
- 删除失败时展示错误提示。

### 7.3 回归测试

- 创建知识库仍可用。
- 上传文档仍可用。
- 文档入库状态轮询仍可用。
- 查看切片仍可用。
- 修改切块配置并重新入库仍可用。
- 问答检索不会命中已删除文档切片。

## 9. 验收标准

- 用户可在知识库界面删除有权限管理的知识库。
- 用户可在知识库文档列表删除有权限管理的文件。
- 删除知识库后，该知识库下文档、解析块、向量切片不再存在。
- 删除文档后，该文档对应解析块、向量切片不再存在。
- 删除后的文件不会再被 RAG 检索命中。
- 删除操作有二次确认、成功提示和失败提示。
- 无权限用户无法通过前端或直接调用 API 删除资源。

---

## 10. 待确认事项

- 删除是否采用硬删除，还是先做软删除并支持恢复？本计划按硬删除设计。
- 对象存储删除失败时，是整体失败回滚，还是数据库先删除并进入异步清理队列？本计划建议第一阶段整体失败并提示。
- 入库中的文档被删除后，Worker 任务是否需要显式撤销？本计划建议先通过“文档不存在则跳过”保证安全，后续再做任务撤销。
