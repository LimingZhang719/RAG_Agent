# P3 知识库与文档入库 研发计划

## 1. 目标与范围

P3 目标是实现知识库与文档入库的可演示闭环：用户可创建知识库、上传文档，系统完成解析、切片、向量化并入库，前端可查看处理状态并支持失败重试。

范围包含：

- 知识库 CRUD（公司级、部门级、个人级）。
- 文档上传与 MinIO 存储。
- 文档解析（PDF/Word/Markdown/TXT/Excel/CSV）。
- Chunk 生成与向量化（Embedding API）。
- 入库任务与状态流转。
- 前端知识库与文档状态展示。
- 前端支持查看文档切片详情。

不在本阶段实现：

- LlamaIndex 问答与 Rerank（P4）。
- 强规则原文返回（P5）。
- Agent 流程（P6+）。

## 2. 关键里程碑

- M1: 数据模型与迁移补齐（如需）。
- M2: 文档上传与对象存储落地。
- M3: 文档解析与切片闭环。
- M4: 向量化与入库任务链路。
- M5: 前端状态展示与重试。
- M6: 基础测试与验收。

## 3. 后端开发计划

### 3.1 数据模型与状态流转

- 确认已有模型：`knowledge_bases`、`documents`、`document_blocks`、`chunks`。
- 新增或补齐字段（如缺失）：
  - `documents.status`：pending / parsing / chunking / embedding / ready / failed
  - `documents.error_message`
  - `chunks` 必备字段：`kb_id`、`document_id`、`content`、`embedding`、`page_no`、`block_order`、`section_path`、`visibility_scope`、`org_id`、`owner_id`、`is_deterministic_rule`
- 如模型不全，新增 Alembic migration。

### 3.2 知识库 API

- CRUD：创建、列表、详情、更新、删除。
- 级别控制：公司级 / 部门级 / 个人级。
- 组织级别：部门知识库必须绑定 `org_id`。
- 权限：创建和查看仅做基础检查（细粒度在 P5）。

### 3.3 文档上传与存储

- 上传接口：`POST /api/documents/upload`
  - 入参：`kb_id`、文件、可选 `title`
  - 行为：
    1. 保存到 MinIO
    2. 创建 `documents` 记录，状态 `pending`
    3. 返回文档信息
- 存储适配：在 `storage/` 增加 MinIO 客户端封装（如果已有则扩展）。
- 文件元数据：保存文件名、大小、类型、对象存储路径。

### 3.4 文档解析与切片

- 统一改为 LlamaIndex 文档加载与切片流程，避免自建解析器，确保与 P4 阶段对接一致。
- 文档加载：使用 LlamaIndex Reader / Loader 读取不同文件类型。
- 切片：使用 LlamaIndex 内置 NodeParser / SentenceSplitter。
- 解析输出：直接生成 LlamaIndex Nodes，并映射为 `document_blocks` + `chunks` 记录。
- 元数据：在 Node metadata 中保留 `page_no`、`block_order`、`section_path`、`document_id`、`kb_id`。

### 3.5 向量化与入库

- 统一使用 LlamaIndex Embedding 抽象与 VectorStore 写入。
- Embedding 配置通过模型适配层注入，避免业务代码依赖具体厂商 SDK。
- pgvector 写入：使用 LlamaIndex 的 PgVectorStore 统一落库。

### 3.6 异步任务

- Celery 任务链：
  - `ingest_document`（LlamaIndex：load → parse → split → embed → vector store）
- 状态流转与失败处理：
  - 任何失败：`documents.status = failed` 且记录 `error_message`
  - 重试：`POST /api/documents/{id}/retry`

### 3.7 状态查询

- 文档列表：返回 `status`、`progress`（可选）、`error_message`。
- 文档详情：返回解析和切片信息概览。
- 文档切片列表：返回每个 chunk 的序号、页码、内容与路径信息。

## 4. 前端开发计划

### 4.1 知识库页面

- 知识库列表与创建表单。
- 创建时选择级别，部门级需选部门。

### 4.2 文档上传组件

- 绑定到知识库详情页。
- 展示上传队列、进度和状态。
- 支持失败重试。

### 4.3 文档状态展示

- 状态标签：pending / parsing / chunking / embedding / ready / failed。
- 失败提示：显示 `error_message`。
- 切片查看：支持查看单文档的 chunk 列表。

## 5. 接口清单（草案）

- `POST /api/knowledge-bases`
- `GET /api/knowledge-bases`
- `GET /api/knowledge-bases/{id}`
- `PATCH /api/knowledge-bases/{id}`
- `DELETE /api/knowledge-bases/{id}`

- `POST /api/documents/upload`
- `GET /api/documents?kb_id=`
- `GET /api/documents/{id}`
- `POST /api/documents/{id}/retry`
- `GET /api/documents/{id}/chunks`

## 6. 数据验收标准

- 通过上传 PDF/Word/Markdown/TXT/Excel/CSV，能完成解析与向量入库。
- 文档状态流转正确，失败有原因并可重试。
- chunks 包含所需字段，向量可写入 pgvector。

## 7. 风险与处理

- 文档解析质量不稳定：保留原文与解析产物，支持单文档重跑。
- 外部 Embedding API 不稳定：增加超时、重试、降级。
- 大文件处理：限制单文件大小与分批处理。

## 8. 研发顺序建议

1. 复核数据模型与 Alembic。
2. 完成 MinIO 客户端与上传接口。
3. 实现解析器与统一输出结构。
4. 实现 chunker 与向量化。
5. 接入 Celery 任务链。
6. 前端上传与状态展示。
7. 验收与修复。

## 9. 待确认事项

- Embedding API 供应商与调用方式。
- 是否先启用 Celery（或同步执行以快速验收）。
- 文档大小与批量限制阈值。
- 文档解析失败是否需要保留中间产物。
