# 智能 Agentic-RAG 知识检索与业务处理平台需求文档

## 1. 项目概述

本项目旨在构建一个高准确率的 **RAG（检索增强生成）知识库检索系统**，并在此基础上演进为 **Agentic RAG（智能体化知识检索）架构**。系统不仅提供精准的企业级知识问答，还将核心业务场景（如报销、入职）沉淀为专属智能体，旨在利用AI自动化解决 99% 的复杂业务流程。

---

## 2. 核心功能需求

### 2.1 文档解析与处理能力

系统需具备强大的多模态、多格式文档解析能力，确保知识入库的高质量：

* **支持格式广泛**：支持识别与解析纯文本、Word、PDF、表格数据（Excel/CSV）以及图像类文件。
* **复杂元素提取**：必须能够精准识别和还原文档中的表格结构与图像内容。
* **抗干扰处理（水印去除）**：针对带有企业或平台水印的 PDF 文件，需具备图像预处理或水印过滤算法，保证底层文本解析的纯净度与准确率。
* **高精度切片与向量化**：准确地将内部文档进行语义分块（Chunking）、解析并转化为高质量向量数据。

### 2.2 RAG 召回与生成策略

RAG 的核心目标是“务必保证整个召回的准确率”，并在生成端进行严格管控：

* **精准召回排序**：检索召回的内容段落顺序必须与原文档逻辑保持完全一致，严禁出现乱序或错位现象。
* **确定性知识防篡改（0幻觉）**：对于 100% 确定的强规则内容（如“十不准原则”、安全规范等），系统需判定为精确匹配，**直接原样返回原文内容**，禁止大模型对其进行任何二次修饰、润色或总结。

### 2.3 权限与知识库层级管理

系统采用严格的 RBAC（基于角色的访问控制）机制，知识库实行三级隔离与可见性保障：

| 知识库级别 | 权限与可见性说明 | 适用场景与操作权限 |
| --- | --- | --- |
| **全公司级** | 全员可见。 | 存放公司规章制度、通用政策。管理员维护，用户仅可检索对话。 |
| **组织/部门级** | 仅当前组织架构内成员可见。 | 存放部门内部文档、业务SOP。根据用户登录身份自动匹配可见范围。 |
| **个人级** | 仅创建者本人可见。 | 用户可自行上传内容、创建私有知识库并进行对话问答，数据完全隔离。 |

## 3. 用户交互与前端展示流程

### 3.1 基础问答与系统闭环

1. **统一鉴权登录**：用户登录系统，系统获取用户身份及组织架构信息。
2. **权限路由渲染**：根据用户信息，动态渲染其有权访问的“公司级”、“部门级”及“个人级”知识库列表。
3. **对话与管理**：用户可选择特定知识库进行靶向提问；也可进入个人空间，自主整理、上传文档创建个人知识库。

### 3.2 首页热门话题（数据看板）

* **热门聚类聚合**：在首页展示全系统内用户高频提问的“热门话题”。
* **内容合规与去重**：
  * **语义去重**：合并意思相近的提问。
  * **敏感词与意图过滤**：内置合规审核机制（敏感词库+大模型意图判定），拦截并屏蔽涉及隐私、八卦或不合规的敏感问题（例如：“领导今年几岁”、“某员工薪资多少”等），禁止其在首页展示。

## 4. 智能体 (AI Agents) 需求

系统需内置专属业务智能体，处理企业内高频复杂流程，目标是覆盖并自动化解决 99% 的常见问题。首期规划以下两类核心智能体：

1. **新员工入职智能体**：处理入职指引、权限申请、办公IT配置等相关操作流程。
2. **报销助手智能体**（详见 `4.1` 深度规划）。

### 4.1 报销助手智能体专项流程设计

该智能体需承担从前端材料收集到后端审批辅助的全链路职责。

* **阶段一：员工申报端（界面与上传）**
* 提供友好的交互界面，引导员工上传发票、水单、审批单等报销凭证。


* **阶段二：智能初审（识别与判断）**
* **材料核验**：OCR识别上传凭证，与报销规则交叉比对。
* **缺漏打回**：若判断材料不全（如缺发票、金额对不上），立即在对话框内返回具体缺漏项，指导用户补充上传。


* **阶段三：流程流转（流程与提交）**
* 材料齐备且初审无误后，智能体将报销单据及解析数据打包，自动流转至财务（报销专员）的待办节点。


* **阶段四：财务审批端（报销人员视角）**
* **结构化看板**：财务人员打开界面，可一目了然地查看结构化的报销明细和原始附件。
* **AI 辅助决策结论**：智能体需给出明确的审核建议（确切回复）：
* 🟢 *推荐通过*：“我认为各项材料合规，可以报销。”
* 🟡 *重点关注*：“发票连号/抬头异常，建议着重检查【某张发票】。”
* 🔴 *风险预警*：“报销金额超出差旅标准，此处存在违规问题。”




* **阶段五：人机协同与资金结算（后续操作）**
* 财务人员无需逐行对账，只需根据 AI 提示的“可疑问题”进行定向复核。
* 人工确认无误并点击通过后，系统进行最终状态校验，随后进入线下/三方系统的人工打款操作设计环节。

## 5. 架构演进：Agentic RAG 实现方案

针对“如何处理复杂问题”及“如何实现智能体化知识检索”的需求，系统核心架构支持从**基础 RAG（检索+生成）** 升级为 **Agentic RAG（智能体 RAG）**。
